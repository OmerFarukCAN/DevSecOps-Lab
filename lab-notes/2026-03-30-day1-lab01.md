## 🐛 Errors and Solutions

### Error 1: Ubuntu Didn't Launch After WSL Install
**Problem:**
```
WSL installed but Ubuntu didn't automatically open
No user creation prompt appeared
```

**Why it happened:**
- WSL2 framework installed but Linux distribution wasn't automatically installed

**How I solved it:**
- Checked available distributions: `wsl --list --online`
- Manually installed Ubuntu 22.04 from Microsoft Store
- Launched Ubuntu and completed initial setup

**What I learned:**
- WSL2 and Linux distributions are separate installations
- Manual installation via Microsoft Store is more reliable

### Error 2: Permission Denied with sudo and Redirection
**Problem:**
```bash
sudo echo "Hello" > file.txt
# Result: Permission denied
```

**Why it happened:**
- Redirection operator (`>`) runs in user context, not sudo context
- Shell processes redirection before sudo executes

**How I solved it:**
- Used `tee` command: `echo "Hello" | sudo tee file.txt`
- Alternative: `sudo sh -c 'echo "Hello" > file.txt'`

**What I learned:**
- Understanding how shell processes commands
- Proper use of sudo with redirection
- tee command for writing with elevated privileges

### Error 3: Working in Wrong Directory
**Problem:**
- Created labs folder in `/home/labs` (required sudo for everything)

**How I solved it:**
- Removed wrong directory: `sudo rm -r /home/labs`
- Created in home directory: `cd ~ && mkdir labs`
- Learned proper directory ownership

**What I learned:**
- Home directory ownership and permissions
- Best practice: work in own home directory (`~`)
- Understanding Linux directory structure

## 📝 Notes

### Technical Notes
```bash
# Most used commands today:
pwd                    # Print working directory
ls -la                 # List all files with details
cd ~                   # Go to home directory
mkdir <dirname>        # Create directory
touch <filename>       # Create empty file
cat <file>             # Display file contents
nano <file>            # Edit file
man <command>          # Read manual

# File operations:
cp source dest         # Copy
mv old new             # Move/rename
rm file                # Remove file
rm -r directory        # Remove directory recursively

# sudo usage:
sudo <command>         # Run with root privileges
sudo usermod -aG docker $USER  # Add user to docker group
```

### General Notes
- Linux uses forward slashes (/) not backslashes (\)
- Commands are case-sensitive
- man pages are incredibly helpful
- ~ means home directory (/home/username)
- . means current directory
- .. means parent directory
